#시멘틱 태그(semantic tag: 이름만 봐도 의미를 알 수 있다)

paragraph(단락)

:anchor #시멘틱 태그 사용 목적 :웹 문서의 구조를 만드는 주요 태그에서 사용 :HTML 소스 코드만 보고도 어느 :문서 구조가 정확히 나눠지기 때문에 PC나 mobile 웹브라우저 다양한 화면에서 웹문서를 표현하기 용의하다 :인터넷에서 웹사이트를 검색할 때 필요한 내용을 정확히 찾을 수 있다 :접근성 향상 seo 최적화, 코드의 가독성 및 유지보수 용의성 #웹 문서 구조를 만드는 시맨틱 태그의 종류
header태그
:해더 영역을 담당 :주로 맨 위쪽이나 왼쪽에 위치를:검색, 사이트 메뉴를 : 태그와 혼동x 웹문서의 정보를 담는 태그
nav 태그
:네비게이션 영역을 담당 :긑은 웹문서에서 다른 위치로, 다른 웹페이지 링크
main 태그
:main콘탠츠 영역 :메뉴, 사이트 ui :웹 문서마다 다르게 보여지는 내용으로 구성 :웹 문서에서 한 번만 사용
article태그
:웹에서 실제로 보여주고 싶은 내용을 삽이하는 영역 :웹문서에서 여러개 사용 가능 :article 태그 안에 section태그 사용 :블로그의 포스트나 뉴스 사이트의 기사처럼 독립된 웹 콘텐츠 항목 같은 내용 삽입
section태그
:콘텐츠 영역 :몇 개의 콘텐츠를 묶는 용도로 사용 :div태그: 단순히 원하는 내용들을 묶을 때 묶는 용도로 사용
aside태그
:사이드바 영역 :본문 내용 외 왼쪽, 오른쪽 혹은 아래쪽에 위치 :웹 사이트의 필수 요소 x
footer태그
:footer 영역 :웹문서 맨 아래쪽에 위치 :사이트 정보, 저작권 정보
div태그
:division: 구분, 분할 :
:
해당영역에 의미를 붙여 사용 :문서 구조(영역구별)를 지정
